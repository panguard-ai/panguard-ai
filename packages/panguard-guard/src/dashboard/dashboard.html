Threat Cloud
Threat Cloud is PanGuard's anonymous threat-intelligence network. Your endpoint uploads anonymized threat patterns (no code, no file contents). New detection rules ship in the open ATR package — Guard checks once a day and tells you when an upgrade is available. It never auto-pulls rule logic from the network, so a compromised relay can't change what your engine enforces.

Flywheel
Your endpoint detects a threat → Threat Cloud aggregates it anonymously → it strengthens the open ATR ruleset → you get the improvements through pga upgrade (a daily check flags when new rules are out).

Anonymous Upload

When enabled, anonymized threat patterns are uploaded to Threat Cloud. No personal data, file contents, or source code is sent.

Settings
Configure how Guard runs on this endpoint.

Guard Mode

Three modes. Learning builds a baseline and only logs. Report-only detects and records everything but never acts (recommended for new deployments). Protection actively blocks per your enforcement policy. Saving a change restarts detection with the new mode.

Threat Cloud Endpoint

URL of the Threat Cloud relay. Leave empty for the default panguard.ai endpoint, or specify your own air-gapped relay.

Dashboard Port

The TCP port this dashboard listens on (localhost-only). Change it with pga config --dashboard-port <port>, then restart Guard.
--

Alert Channels

Where Guard sends an alert when a threat fires. Configured with pga config / ~/.panguard-guard/config.json — secrets are never shown or set from this dashboard.

Enforcement Policy — what Protection mode is allowed to do

In Protection mode, only the responses armed below run automatically. Configure under enforcementPolicy in ~/.panguard-guard/config.json.
