Get Panguard running, scan your AI agent skills for threats, and start real-time protection. No account required. Works on macOS, Linux, and Windows.

<Steps>
  <Step title="Install">
    <Tabs>
      <Tab title="macOS / Linux">
        ```bash
        curl -fsSL https://get.panguard.ai | bash
        ```
      </Tab>
      <Tab title="Windows">
        ```powershell
        powershell -ExecutionPolicy Bypass -Command "irm https://get.panguard.ai/windows | iex"
        ```
      </Tab>
      <Tab title="npm (all platforms)">
        ```bash
        npm install -g @panguard-ai/panguard
        ```
      </Tab>
    </Tabs>

    Verify:

    ```bash
    panguard --version
    # Expected: 1.0.0
    ```
  </Step>

  <Step title="Connect your AI agents">
    Auto-detect and configure all AI platforms on your machine:

    ```bash
    panguard setup
    ```

    This finds Claude Code, Claude Desktop, Cursor, OpenClaw, Codex, WorkBuddy, NemoClaw, ArkClaw and configures MCP integration automatically:

    ```
    PANGUARD AI - Setup

    Detected platforms:
      Claude Code    ~/.claude/settings.local.json
      Cursor         ~/.cursor/mcp.json
      OpenClaw       ~/.openclaw/skills/panguard/SKILL.md

    All platforms configured.
    ```

    <Info>
      No platforms detected? You can still use Panguard directly from the CLI. Skip to the next step.
    </Info>
  </Step>

  <Step title="Scan your AI agent skills">
    Audit every MCP skill and AI agent tool installed on your system:

    ```bash
    panguard-guard scan
    ```

    The scan checks all installed skills against 61 ATR (Agent Threat Rules) covering prompt injection, tool poisoning, credential theft, and 6 more threat categories:

    ```
    PANGUARD [#] AI  v1.0.0

    Scanning: claude-code, claude-desktop, cursor, openclaw, codex, workbuddy, nemoclaw, arkclaw

    Results:
      totalSkills: 23
      CLEAN:    18
      MEDIUM:    3
      HIGH:      1
      CRITICAL:  1

    highestRisk: HIGH
    ```

    Add `--json` for machine-readable output that AI agents can parse directly.

    <Warning>
      **CRITICAL and HIGH findings need attention.** Review each finding and decide whether to keep, update, or remove the flagged skill. Panguard never auto-removes anything.
    </Warning>
  </Step>

  <Step title="Start real-time protection">
    A scan is a point-in-time check. For continuous monitoring, start Guard:

    ```bash
    panguard-guard start
    ```

    Guard runs in the background and watches all AI agent activity:

    ```
    PANGUARD [#] AI  v1.0.0

    Guard engine started.

    Status:     RUNNING
    Mode:       learning (Day 1/7)
    Rules:      61 ATR detection rules
    Dashboard:  http://localhost:3100
    ```

    <Info>
      **Learning mode (Days 1-7):** Guard observes normal behavior and builds a baseline. No false positives during this period. After Day 7, anomaly detection activates automatically.
    </Info>

    Other commands:

    ```bash
    panguard-guard status    # check if running
    panguard-guard stop      # stop the daemon
    ```
  </Step>
</Steps>

## Two CLIs, different purposes

| Command | What it does |
| --- | --- |
| `panguard scan` | System-level security scan (ports, firewall, services) |
| `panguard-guard scan` | AI agent skill scan (MCP tools, prompt injection, supply chain) |
| `panguard setup` | Auto-configure AI platforms |
| `panguard-guard start` | Start real-time AI agent monitoring |

Most users want `panguard-guard scan` first (check your AI tools), then `panguard-guard start` (keep them monitored).

## How collective defense works

When you run Panguard, your machine joins a collective defense network:

1. **Guard detects a threat** on your machine.
2. **An anonymous hash is shared** with Threat Cloud (no personal data, no source code).
3. **3 independent scanners confirm** the same pattern — a new detection rule is auto-generated.
4. **All Panguard users get the rule** within 1 hour.

One machine gets attacked. One hour later, every machine is immune.
