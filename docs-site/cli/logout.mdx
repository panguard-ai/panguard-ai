---
title: "panguard logout"
description: "Disconnect from Panguard Cloud and clear stored credentials."

---

The `logout` command disconnects your CLI from Panguard Cloud by removing the stored access token. Core security features continue to work without authentication.

## Usage

```bash
panguard logout [options]
```

## Options

<ParamField path="--lang" type="string" default="en">
  Set the language for logout messages.
</ParamField>

## Examples

<CodeGroup>
```bash Disconnect from cloud
panguard logout
```

```bash Disconnect with Chinese messages
panguard logout --lang zh-TW
```
</CodeGroup>

## What It Does

1. **Removes the local token** stored in `~/.panguard/credentials`
2. **Notifies the server** to revoke the session (if reachable)
3. **Clears cached cloud data** from the local configuration

<Info>
  After logging out, all core features (Guard, Skill Auditor, Threat Cloud, ATR rules) continue to work. Only cloud-specific features like multi-endpoint management are disabled.
</Info>
