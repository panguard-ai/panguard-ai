The `login` command connects your CLI to Panguard Cloud for optional enhanced features like multi-endpoint management and cloud AI analysis. **Login is not required** -- all core features (Guard, Skill Auditor, ATR rules, Threat Cloud sync) work without authentication.

## Usage

```bash
panguard login [options]
```

## Options

<ParamField path="--api-url" type="string">
  Override the default Panguard API endpoint. Useful for self-hosted or staging environments.
</ParamField>

<ParamField path="--no-browser" type="flag">
  Use device-code authentication instead of opening a browser. Prints a URL and code to enter manually -- ideal for SSH sessions and headless servers.
</ParamField>

<ParamField path="--lang" type="string" default="en">
  Set the language for login prompts and messages.
</ParamField>

## Examples

<CodeGroup>
```bash Browser-based login (default)
panguard login
```

```bash Headless / SSH login
panguard login --no-browser
```

```bash Self-hosted API endpoint
panguard login --api-url https://panguard.internal.example.com/api
```

```bash Login in Traditional Chinese
panguard login --lang zh-TW
```
</CodeGroup>

<Info>
  **Login is optional.** Panguard is 100% functional without authentication. Login unlocks cloud features like centralized multi-endpoint management. All security features -- Guard, Skill Auditor, Threat Cloud, ATR rules -- work offline and without an account.
</Info>
