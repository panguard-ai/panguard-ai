`doctor` 跑一系列診斷檢查，幫你找出 Panguard 安裝、config、運行環境的常見問題。每個發現都會附上具體的處理建議。

## 用法

```bash
panguard doctor
```

## 診斷項目

doctor 會檢查這些範圍：

| 項目         | 檢查什麼                                       |
| ------------ | ---------------------------------------------- |
| **安裝**     | binary 完整性、版本是不是最新、相依套件        |
| **設定**     | config 檔格式對不對、必填欄位、路徑權限        |
| **Guard**    | 服務狀態、log 檔健康狀態、資源用量             |
| **網路**     | 連到 Threat Cloud 的連線、DNS 解析、防火牆規則 |
| **權限**     | 檔案擁有者、目錄存取權、是否需要提權           |
| **磁碟空間** | log、掃描資料、情報資料庫還有多少空間          |

```
Panguard Doctor v1.5.6
======================

[PASS] Installation: binary intact, version 1.5.6 (latest)
[PASS] Configuration: ~/.panguard/config.yaml valid
[WARN] Guard: not running -- start with 'panguard guard start'
[PASS] Network: cloud API reachable (latency 45ms)
[PASS] Permissions: all directories accessible
[FAIL] Disk Space: /var/log/panguard has only 200MB free (minimum 500MB)

1 failure, 1 warning, 4 passed
```

<Tip>
  遇到怪問題的時候，先跑 `panguard doctor` 再回報 issue。輸出格式設計成可以直接貼給支援團隊看。
</Tip>
