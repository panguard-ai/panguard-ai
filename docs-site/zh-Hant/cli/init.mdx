`init` 在你的機器上建好 Panguard 環境。它會產生預設 config 檔、建資料目錄，自動偵測你的 OS 跟裝了什麼服務來設最佳預設值。

## 用法

```bash
panguard init [options]
```

## 選項

<ParamField path="--config" type="string">
  自訂 config 檔路徑。如果有給，`init` 會把你的自訂設定跟偵測到的預設值合併，不會從零開始建。
</ParamField>

<ParamField path="--lang" type="string" default="en">
  設定 CLI 輸出跟通知的預設語言。
</ParamField>

## 範例

<CodeGroup>
```bash 用預設值初始化
panguard init
```

```bash 用自訂 config 初始化
panguard init --config ./my-panguard.yaml
```

```bash 用繁體中文初始化
panguard init --lang zh-TW
```
</CodeGroup>

## `init` 做了什麼

1. **偵測 OS 跟環境** -- 辨識 Linux 發行版、macOS 版本或 WSL 設定
2. **建 config 目錄** -- `~/.panguard/` 加上預設的 `config.yaml`
3. **建資料目錄** -- Guard 跟 Scan 各自獨立的資料目錄
4. **偵測跑著的服務** -- 辨識 web server、資料庫之類的服務，調整掃描跟防護的預設值
5. **設定 Threat Cloud** -- 問你要不要啟用集體防禦（匿名威脅資料共享）

<Tip>
  每台機器只需要跑一次 `init`。重跑是安全的 -- 除非你明確給新的 config 檔，否則現有設定不會被蓋掉。
</Tip>
