`trap` 指令部署輕量級蜜罐服務，模擬真實的網路服務（SSH、HTTP、SMB 等）。當攻擊者與蜜罐互動時，Panguard 會擷取其戰術、產生威脅情報，並立即向您發出警報。

## 使用方式

```bash
panguard trap <subcommand> [options]
```

## 子指令

| 子指令     | 說明                               |
| ---------- | ---------------------------------- |
| `start`    | 啟動蜜罐服務                       |
| `stop`     | 停止所有運行中的蜜罐服務           |
| `status`   | 顯示活躍的蜜罐和互動統計           |
| `config`   | 檢視或修改蜜罐設定                 |
| `profiles` | 列出從蜜罐互動中建立的攻擊者描述檔 |
| `intel`    | 匯出收集到的威脅情報資料           |

## 選項

<ParamField path="--services" type="string">
  以逗號分隔的蜜罐服務類型清單。選項包括 `ssh`、`http`、`smb`、`ftp`、`mysql`、`redis`、`rdp`。
</ParamField>

<ParamField path="--data-dir" type="string">
  覆寫蜜罐日誌和情報資料的預設目錄。預設為 `~/.panguard/trap/`。
</ParamField>

<ParamField path="--no-cloud" type="flag">
  將所有擷取的情報保留在本機 -- 不上傳至 Panguard Threat Cloud。
</ParamField>

## 範例

<CodeGroup>
```bash 啟動 SSH 和 HTTP 蜜罐
panguard trap start --services ssh,http
```

```bash 啟動所有預設蜜罐
panguard trap start
```

```bash 檢查蜜罐狀態和統計
panguard trap status
```

```bash 檢視攻擊者描述檔
panguard trap profiles
```

```bash 匯出威脅情報
panguard trap intel
```

```bash 離線模式（不上傳至雲端）
panguard trap start --services ssh,http --no-cloud
```

</CodeGroup>

## 支援的蜜罐服務

| 服務    | 預設埠 | 模擬對象                |
| ------- | ------ | ----------------------- |
| `ssh`   | 2222   | OpenSSH 伺服器          |
| `http`  | 8080   | Apache/Nginx 網頁伺服器 |
| `smb`   | 4450   | Windows 檔案共享        |
| `ftp`   | 2121   | FTP 伺服器              |
| `mysql` | 3307   | MySQL 資料庫            |
| `redis` | 6380   | Redis 鍵值儲存          |
| `rdp`   | 3390   | Windows 遠端桌面        |
