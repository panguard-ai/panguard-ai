`report` 從你的 scan 和 Guard 資料產生專業合規報告。可以對應 ISO 27001、SOC 2、NIST CSF、台灣 TCSA 等框架，稽核準備直接省一半的工。

## 用法

```bash
panguard report <subcommand> [options]
```

## 子指令

| 子指令            | 說明                             |
| ----------------- | -------------------------------- |
| `generate`        | 產生合規報告                     |
| `summary`         | 顯示簡要合規摘要，不產生完整報告 |
| `list-frameworks` | 列出所有支援的合規框架           |
| `validate`        | 驗證已有報告的完整性跟正確性     |
| `config`          | 查看或修改報告設定               |

## 選項

<ParamField path="--framework" type="string">
  目標合規框架。跑 `panguard report list-frameworks` 看所有選項。
</ParamField>

<ParamField path="--language" type="string" default="en">
  報告內容語言（例如 `en`、`zh-TW`）。
</ParamField>

<ParamField path="--format" type="string" default="pdf">
  輸出格式：`pdf`、`html`、`json` 或 `csv`。
</ParamField>

<ParamField path="--output-dir" type="string">
  報告輸出目錄。預設是目前工作目錄。
</ParamField>

<ParamField path="--org" type="string">
  要放在報告標頭跟 metadata 裡的組織名稱。
</ParamField>

## 範例

<CodeGroup>
```bash 產生 ISO 27001 報告
panguard report generate --framework iso27001 --org "Acme Corp"
```

```bash 產生中文 SOC 2 報告
panguard report generate --framework soc2 --language zh-TW
```

```bash 列出可用框架
panguard report list-frameworks
```

```bash 快速合規摘要
panguard report summary --framework nist-csf
```

```bash 匯出 HTML
panguard report generate --framework tcsa --format html --output-dir ./reports
```

```bash 驗證現有報告
panguard report validate --output-dir ./reports
```

</CodeGroup>

## 支援的框架

| 框架         | 識別碼     | 說明                   |
| ------------ | ---------- | ---------------------- |
| ISO 27001    | `iso27001` | 國際資訊安全標準       |
| SOC 2        | `soc2`     | 服務組織信任準則       |
| NIST CSF     | `nist-csf` | NIST 網路安全框架      |
| CIS Controls | `cis`      | CIS 關鍵安全控制       |
| Taiwan TCSA  | `tcsa`     | 台灣企業安全聯盟標準   |
| PCI DSS      | `pci-dss`  | 支付卡產業資料安全標準 |
