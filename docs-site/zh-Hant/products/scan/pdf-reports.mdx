Panguard Scan 能產出專業 PDF 報告，拿來給主管看、附在合規文件裡、或當稽核證據都沒問題。

## 產出報告

```bash
# 英文報告（預設）
panguard scan --output report.pdf

# 繁體中文報告
panguard scan --output report.pdf --lang zh-TW

# 快速掃描配報告
panguard scan --quick --output quick-report.pdf
```

## 報告結構

每份 PDF 報告有五個章節：

### 1. 封面

| 元素          | 說明                             |
| ------------- | -------------------------------- |
| 組織名稱      | 用 `--org` flag 或 config 檔設定 |
| 掃描日期時間  | 掃描當下的 UTC timestamp         |
| Panguard 版本 | 產出報告的 CLI 版本              |
| 品牌 logo     | Panguard AI branding             |

### 2. 執行摘要

給非技術主管看的一頁重點：

- **風險分數**：0--100 安全分數配字母等級
- **弱點統計**：按嚴重度分類（Critical / High / Medium / Low / Info）
- **關鍵指標**：開放 port 數、執行中服務數、偵測到的安全工具
- **總體評估**：用白話總結系統安全狀態

### 3. 弱點明細表

每個弱點一行，欄位如下：

| 欄位       | 說明                                          |
| ---------- | --------------------------------------------- |
| **#**      | 流水號                                        |
| **嚴重度** | CRITICAL、HIGH、MEDIUM、LOW、INFO，有顏色標示 |
| **標題**   | 弱點簡述                                      |
| **類別**   | 掃描器類別（網路、認證、加密等）              |
| **描述**   | 風險的詳細說明                                |
| **位置**   | 受影響的資源（port、檔案路徑、服務名稱）      |

### 4. 修復建議

每個弱點都附：

- 一步一步的修復指引
- 依平台列出指令（Linux、macOS、Windows）
- 相關文件連結
- 依嚴重度和工作量排優先序

### 5. 合規對應

每個弱點對應到合規框架的控制項：

| 框架      | 控制項參考                       |
| --------- | -------------------------------- |
| ISO 27001 | 例如 A.12.6.1 -- 技術弱點管理    |
| SOC 2     | 例如 CC6.1 -- 邏輯與實體存取控制 |
| 台灣 TCSA | 例如第 4 節 -- 存取控制          |

## 雙語支援

報告支援兩種語言。語言設定會影響報告裡所有文字，包括章節標題、弱點描述、修復步驟。

| Flag           | 語言         |
| -------------- | ------------ |
| `--lang en`    | 英文（預設） |
| `--lang zh-TW` | 繁體中文     |

## 遠端掃描報告

遠端目標掃描也能產出報告：

```bash
panguard scan --target 203.0.113.50 --output remote-report.pdf
```

遠端掃描報告會標出目標 hostname/IP，並且註明哪些掃描器可以遠端跑、哪些需要本機才行。

## 整合合規報告

Scan 的 PDF 報告可以當作合規報告的輸入證據（透過 Scan 的 `panguard report` 指令）：

```bash
# 產出掃描報告
panguard scan --output scan-findings.pdf

# 拿弱點當輸入，產出合規報告
panguard report generate --framework iso27001 --format pdf
```

<Info>PDF 產出用的是 server-side rendering，不需要額外裝 PDF 工具或瀏覽器。</Info>
