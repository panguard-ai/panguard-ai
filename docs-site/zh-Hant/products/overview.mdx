Panguard AI 不是單一工具，而是一整套互相搭配的安全產品，各自負責安全生命週期的不同階段。串起來就是一個完整的防護迴圈：掃描系統、即時監控、通知對的人、產出合規報告，再跨機器共享威脅情報。

## 產品怎麼串在一起

這些產品設計上就是要互相搭配，形成分層防禦：

```
Scan ──> Guard ──> Threat Cloud
```

| 使用情境              | 涉及產品             | 說明                                                                                          |
| --------------------- | -------------------- | --------------------------------------------------------------------------------------------- |
| **評估 + 防護**       | Scan + Guard         | 先跑一次掃描建立 baseline，再開 Guard 持續監控。                                              |
| **偵測 + 通知**       | Guard                | Guard 即時偵測威脅，用白話通知送到你指定的頻道（Telegram、Slack、Email、Webhook）。           |
| **合規 + 報告**       | Scan                 | 掃描結果直接對應合規控制項。用 `panguard scan --output report.pdf` 就能產出可交付稽核的 PDF。 |
| **規模化 + 集中管理** | Guard + Threat Cloud | 在多台機器部署 Guard，透過 Threat Cloud 做跨機器的威脅關聯。                                  |
| **AI 副駕駛**         | MCP + Scan + Guard   | 在 Claude 或 Cursor 裡用自然語言跑掃描、看狀態、封鎖 IP，全部透過 MCP server。                |

## 架構概覽

Panguard 是一個 **13 個 package 的 TypeScript monorepo**，用 pnpm workspaces 管理。所有 package 共用 `@panguard-ai/core` 基底，提供規則引擎、monitor 引擎、AI provider 抽象層、i18n 和結構化 logging。

| 分層     | 組件         | 負責什麼                  |
| -------- | ------------ | ------------------------- |
| **端點** | Guard, Scan  | 本地偵測、掃描、通知      |
| **管理** | Guard        | 端點防護與本地 fleet 協調 |
| **情報** | Threat Cloud | 集體 IoC 共享             |
| **整合** | MCP          | AI 助手整合               |

<Info>每個產品都可以獨立跑。不需要一次部署全套。先從 `panguard scan` 開始，有需要再逐步加。</Info>
