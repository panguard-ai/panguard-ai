攻擊活動代表透過在 Panguard Threat Cloud 網路中關聯 IoC 而識別的協調攻擊模式。當多個指標共享行為模式、基礎設施或時間特徵時，它們會被分組為一個攻擊活動。

## GET /api/campaigns

列出所有已識別的攻擊活動，支援分頁。

<ParamField query="status" type="string">
  依狀態篩選：`active`、`dormant`、`resolved`。
</ParamField>

<ParamField query="since" type="string">
  ISO 8601 時間戳。回傳此時間之後更新的攻擊活動。
</ParamField>

<ParamField query="page" type="number" default="1">
  頁碼。
</ParamField>

<ParamField query="limit" type="number" default="20">
  每頁結果數（最大 100）。
</ParamField>

<CodeGroup>
```bash cURL
curl -X GET "https://tc.panguard.ai/api/campaigns?status=active&limit=10" \
  -H "Authorization: Bearer YOUR_API_KEY"
```
</CodeGroup>

### 攻擊活動狀態

| 狀態 | 說明 |
|--------|-------------|
| `active` | 在過去 7 天內觀測到新的 IoC 或觀測記錄 |
| `dormant` | 7--30 天無新活動，但基礎設施可能仍在運作 |
| `resolved` | 30 天以上無活動，相關基礎設施似乎已停用 |

<Tip>
訂閱 [SSE 事件串流](/api-reference/manager/events#get-apieventsstream) 以在您的代理遇到與活躍攻擊活動相關的 IoC 時獲得即時通知。
</Tip>
