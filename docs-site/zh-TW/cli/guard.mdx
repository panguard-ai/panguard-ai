`guard` 指令控制 Panguard 的即時防護代理 -- 一個三層 AI 管線，持續監控您的系統、偵測威脅並自動回應。Guard 以背景服務運行，並透過您設定的通知頻道回報事件。

## 使用方式

```bash
panguard guard <subcommand> [options]
```

## 子指令

| 子指令 | 說明 |
| --- | --- |
| `start` | 啟動 Guard 代理 |
| `stop` | 停止運行中的 Guard 代理 |
| `status` | 顯示目前 Guard 狀態和統計資料 |
| `install` | 將 Guard 安裝為系統服務（systemd / launchd） |
| `uninstall` | 移除 Guard 系統服務 |
| `config` | 檢視或修改 Guard 設定 |
| `help` | 顯示 Guard 子指令說明 |

## 選項

<ParamField path="--data-dir" type="string">
  覆寫 Guard 儲存日誌、基線和狀態檔案的預設資料目錄。預設為 `~/.panguard/guard/`。
</ParamField>

## 範例

<CodeGroup>
```bash 啟動 Guard 防護
panguard guard start
```

```bash 檢查 Guard 狀態
panguard guard status
```

```bash 安裝為系統服務
panguard guard install
```

```bash 使用自訂資料目錄
panguard guard start --data-dir /opt/panguard/data
```

```bash 停止 Guard
panguard guard stop
```

```bash 檢視 Guard 設定
panguard guard config
```
</CodeGroup>

## 運作原理

Guard 透過三層 AI 處理運作：

1. **第一層 -- 規則引擎** -- ATR 規則，用於已知威脅模式
2. **第二層 -- 行為 AI** -- 機器學習基線偵測異常的程序、網路和檔案活動
3. **第三層 -- LLM 判斷** -- 大型語言模型關聯事件、判斷嚴重程度，並產生白話警報

<Tip>
  啟動 Guard 後，會在前 24 小時進入**學習模式**以建立行為基線。在此期間，您可能會看到較多的資訊性警報。
</Tip>
