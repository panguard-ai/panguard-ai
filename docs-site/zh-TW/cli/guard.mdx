`guard` 控制 Panguard 的即時防護 daemon -- 一條三層 AI 管線，持續監控你的系統、偵測威脅、自動處理。Guard 跑在背景，透過你設定的通知頻道回報事件。

## 用法

```bash
panguard guard <subcommand> [options]
```

## 子指令

| 子指令 | 說明 |
| --- | --- |
| `start` | 啟動 Guard daemon |
| `stop` | 停止 Guard daemon |
| `status` | 顯示 Guard 目前狀態跟統計資訊 |
| `install` | 把 Guard 裝成系統服務（systemd / launchd） |
| `uninstall` | 移除 Guard 系統服務 |
| `config` | 查看或修改 Guard config |
| `help` | 顯示 Guard 子指令說明 |

## 選項

<ParamField path="--data-dir" type="string">
  覆蓋 Guard 存放 log、baseline、狀態檔的預設資料目錄。預設 `~/.panguard/guard/`。
</ParamField>

## 範例

<CodeGroup>
```bash 啟動 Guard
panguard guard start
```

```bash 看 Guard 狀態
panguard guard status
```

```bash 裝成系統服務
panguard guard install
```

```bash 用自訂資料目錄
panguard guard start --data-dir /opt/panguard/data
```

```bash 停 Guard
panguard guard stop
```

```bash 看 Guard config
panguard guard config
```
</CodeGroup>

## 運作原理

Guard 透過三層 AI 處理：

1. **第一層 -- 規則引擎** -- 用 ATR 規則比對已知威脅模式（v1.3.0 內建 61 條規則）
2. **第二層 -- 行為 AI** -- 機器學習建立 baseline，偵測異常的 process、網路跟檔案活動
3. **第三層 -- LLM 判斷** -- 大型語言模型做事件關聯、嚴重性判斷，產生白話警示

<Tip>
  啟動 Guard 後，前 24 小時會進入**學習模式**建立行為 baseline。這段期間你可能會看到比較多資訊性的警示，這是正常的。
</Tip>
