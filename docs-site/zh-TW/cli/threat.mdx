`threat` 在本機跑一個 Panguard Threat Cloud 實例 -- 一台威脅情報伺服器，從你的 Guard daemon 匯整 IoC（入侵指標）、攻擊活動資料跟攻擊者 profile。提供 REST API 跟即時儀表板做威脅分析。

## 用法

```bash
panguard threat <subcommand> [options]
```

## 子指令

| 子指令 | 說明 |
| --- | --- |
| `start` | 啟動 Threat Cloud 伺服器 |
| `stats` | 不啟動伺服器，直接看威脅情報統計 |

## 選項

<ParamField path="--port" type="number" default="9500">
  Threat Cloud API 跟儀表板的 port。
</ParamField>

<ParamField path="--host" type="string" default="127.0.0.1">
  伺服器 bind 的位址。用 `0.0.0.0` 監聽所有 interface。
</ParamField>

<ParamField path="--db" type="string">
  儲存威脅情報的 SQLite 資料庫路徑。預設 `~/.panguard/threat/threat.db`。
</ParamField>

## 範例

<CodeGroup>
```bash 啟動 Threat Cloud
panguard threat start
```

```bash 用自訂 port 啟動
panguard threat start --port 8500 --host 0.0.0.0
```

```bash 看威脅統計
panguard threat stats
```

```bash 指定資料庫路徑
panguard threat start --db /opt/panguard/threat.db
```
</CodeGroup>

## 情報來源

Threat Cloud 從這些地方匯整資料：

| 來源 | 資料類型 |
| --- | --- |
| **Guard daemon** | process 異常、網路指標、檔案 hash |
| **社群 feed** | 公開 IoC feed、STIX/TAXII 來源 |
| **手動上傳** | 透過 API 傳的自訂 IoC 清單 |

<Warning>
  bind 到 `0.0.0.0` 的話，Threat Cloud API 就暴露在網路上了。務必設好認證，放在防火牆或 reverse proxy 後面。
</Warning>
