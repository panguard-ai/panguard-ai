---
title: "panguard deploy"
description: "將 Panguard 設定和代理部署到遠端端點。"
icon: "rocket"
---

`deploy` 指令將您的本機 Panguard 設定推送到遠端機器，實現多端點間一致的安全策略。它在一個步驟中處理代理安裝、設定分發和服務註冊。

## 使用方式

```bash
panguard deploy [options]
```

## 選項

<ParamField path="--config" type="string">
  定義目標主機和策略的部署設定檔路徑。預設為 `~/.panguard/deploy.yaml`。
</ParamField>

<ParamField path="--dry-run" type="flag">
  模擬部署而不進行任何變更。顯示將要部署到哪些主機的內容。
</ParamField>

<ParamField path="--lang" type="string" default="en">
  設定部署日誌的輸出語言。
</ParamField>

## 範例

<CodeGroup>
```bash 使用預設設定部署
panguard deploy
```

```bash 預覽變更的試執行
panguard deploy --dry-run
```

```bash 使用自訂設定部署
panguard deploy --config ./production-deploy.yaml
```

```bash 以中文輸出部署
panguard deploy --lang zh-TW
```
</CodeGroup>

## 部署設定

部署設定檔（`deploy.yaml`）定義目標主機和要套用的策略：

```yaml
targets:
  - host: 192.168.1.10
    user: panguard
    services:
      - guard
      - scan
  - host: web-server.example.com
    user: root
    services:
      - guard
      - trap
      - chat

policy:
  guard:
    auto_response: true
    learning_period: 24h
  scan:
    schedule: "0 2 * * *"  # Daily at 2 AM
```

<Tip>
  在對正式環境系統進行變更之前，請務必先執行 `panguard deploy --dry-run` 來驗證目標清單和設定。
</Tip>

</Info>
