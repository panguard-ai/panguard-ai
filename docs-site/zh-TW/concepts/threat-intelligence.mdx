# 威脅情報

威脅情報提供關於已知攻擊者、惡意 IP、網域、URL 與惡意軟體特徵的結構化資訊。Panguard 自動查詢這些資料庫，以判斷您系統上的活動是否與已知威脅有關。

<Info>
您不需要了解技術細節。Guard 會自動處理查詢，Chat 則以白話語言解釋結果。
</Info>

---

## 5 個內建威脅情報來源

### abuse.ch 套件

| 來源 | 指標類型 | 說明 |
| --- | --- | --- |
| [ThreatFox](https://threatfox.abuse.ch) | IP、域名、URL、雜湊 | 與惡意軟體攻擊活動相關的入侵指標 (IoC) 資料庫 |
| [URLhaus](https://urlhaus.abuse.ch) | URL | 惡意軟體散布 URL 資料庫 |
| [Feodo Tracker](https://feodotracker.abuse.ch) | IP | 殭屍網路命令控制 (C2) 伺服器追蹤 |

### 其他來源

| 來源 | 指標類型 | 說明 |
| --- | --- | --- |
| [GreyNoise](https://greynoise.io) | IP | 區分針對性攻擊與大規模網路掃描 |
| [AbuseIPDB](https://abuseipdb.com) | IP | 社群回報的惡意 IP 資料庫，附帶信心度評分 |

## 來源更新排程

| 來源 | 更新頻率 | 查詢方式 |
| --- | --- | --- |
| ThreatFox | 每小時 | 本地快取 |
| URLhaus | 每小時 | 本地快取 |
| Feodo Tracker | 每小時 | 本地快取 |
| GreyNoise | 每次查詢即時 | API 呼叫 |
| AbuseIPDB | 每次查詢即時 | API 呼叫 |

<Tip>
1 小時的更新間隔可以調整。頻寬受限的系統可延長至 6 或 24 小時。
</Tip>

### 本地快取

查詢結果會在本地快取以避免重複查詢：

- 快取持續時間：依來源 1-24 小時
- 快取位置：Guard 資料目錄
- 過期的項目會自動清除

## 入侵指標 (IoC)

威脅情報追蹤以下類型的指標：

| 類型 | 說明 | 範例 |
| --- | --- | --- |
| **IP 位址** | 已知惡意 IP | `203.0.113.50` |
| **域名** | 惡意域名 | `malware.example.com` |
| **URL** | 惡意 URL | `http://evil.com/payload.exe` |
| **檔案雜湊** | 惡意軟體指紋（SHA-256） | `e3b0c44298fc1c149a...` |
| **電子郵件** | 釣魚電子郵件地址 | `phish@attacker.com` |

### 自動查詢

Guard 偵測到可疑活動時會自動查詢威脅情報：

```
Suspicious IP 203.0.113.50 connection detected
       |
       v
  Query ThreatFox  -> Known C2 server
  Query AbuseIPDB  -> Reported 1,247 times
  Query GreyNoise  -> Mass scanner
       |
       v
  Conclusion: High risk -- auto-block + notify
```

## Threat Cloud -- 集體情報

除了公開來源外，Panguard 使用者還可貢獻並受益於 Threat Cloud，這是一個社群驅動的集體情報網路，專注於 ATR（Agent Threat Rules）共識。

### 結晶化飛輪

Threat Cloud 的核心價值在於結晶化飛輪 -- 一個自我強化的循環，將個別掃描發現項目提煉為社群確認的偵測規則：

```
  掃描技能 ──> 發現項目 ──> TC 提案 ──> 共識（3+ 次確認）──> 已確認規則
       ^                                                              |
       └──────────────── 分發至所有掃描器 ────────────────────────────┘
```

每次循環都強化整個網路：

1. **掃描** -- 任何 Panguard 掃描器（CLI、Website 或 Guard）掃描一個 MCP 技能
2. **提案** -- 高嚴重性發現項目產生以模式雜湊識別的 ATR 提案
3. **確認** -- 其他掃描器遇到相同模式雜湊時確認該提案
4. **升格** -- 達到 3+ 次獨立確認後，提案自動升格為已確認規則
5. **分發** -- 已確認規則透過 `GET /api/atr-rules` 提供給所有掃描器
6. **強化** -- 掃描器載入新規則，提升偵測能力，產生更多提案

模式雜湊使用格式 `scan:{skillName}:{findingSummary}`，以 SHA-256 截斷為 16 個十六進位字元。因為所有掃描器使用相同的 `@panguard-ai/scan-core` 函式庫，無論掃描源自 CLI、Website 或 Guard，都會對相同威脅模式產生相同的雜湊。

### LLM 審查員

Threat Cloud 包含一個自動化 LLM 審查員（Claude Sonnet 4），用於評估 ATR 提案的誤報風險、覆蓋度、偵測專一性和 YAML 有效性。提案可透過僅社群共識（3+ 次確認）或 LLM 核准結合社群確認來升格。

### IoC 來源

Threat Cloud 也分發傳統的 IoC 來源（IP 黑名單、網域黑名單）並維護社群技能黑名單。這些作為 ATR 規則管線在網路層級威脅指標方面的補充。

## 隱私與資料保護

<Warning>
**隱私保證：** 僅上傳威脅指標（IP、雜湊、模式）。絕不分享任何系統資訊、使用者名稱、內部 IP、檔案內容或個人識別資訊。
</Warning>

| 隱私措施 | 詳情 |
| --- | --- |
| **IP 匿名化** | 來源 IP 在上傳前進行 /16 匿名化 |
| **GDPR 合規** | 不收集或儲存個人資料 |
| **零原始資料** | 不傳輸日誌內容、檔案內容或系統詳情 |
| **零遙測** | 不收集使用分析、當機報告或行為追蹤 |
| **可選擇退出** | Threat Cloud 可完全停用 |

## 離線模式

Panguard 可完全離線運作。當威脅情報來源無法連線時：

- Layer 1 規則引擎以本地快取規則持續運作
- 先前快取的來源資料在過期前仍可使用
- 新的偵測僅依賴 ATR 規則與行為基線
- 分數調整會反映情報覆蓋率降低

```bash
# 停用所有外部情報（僅規則模式）
panguard guard start --offline
```

<Note>
離線模式停用 Threat Cloud 參與與即時來源查詢。快取資料在過期前仍會使用。
</Note>
