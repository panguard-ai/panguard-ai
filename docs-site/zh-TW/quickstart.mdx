裝好 Panguard、掃描你的 AI Agent Skill、啟動即時防護。不需帳號。macOS、Linux、Windows 都支援。

<Steps>
  <Step title="安裝">
    <Tabs>
      <Tab title="macOS / Linux">
        ```bash
        curl -fsSL https://get.panguard.ai | bash
        ```
      </Tab>
      <Tab title="Windows">
        ```powershell
        powershell -ExecutionPolicy Bypass -Command "irm https://get.panguard.ai/windows | iex"
        ```
      </Tab>
      <Tab title="npm (全平台)">
        ```bash
        npm install -g @panguard-ai/panguard
        ```
      </Tab>
    </Tabs>

    驗證安裝：

    ```bash
    panguard --version
    # 預期輸出: 1.3.0
    ```
  </Step>

  <Step title="連接 AI Agent">
    自動偵測並設定你機器上所有的 AI 平台：

    ```bash
    panguard setup
    ```

    會自動找到 Claude Code、Claude Desktop、Cursor、OpenClaw、Codex、WorkBuddy、NemoClaw、ArkClaw 並設定 MCP 整合：

    ```
    PANGUARD AI - Setup

    Detected platforms:
      Claude Code    ~/.claude/settings.local.json
      Cursor         ~/.cursor/mcp.json
      OpenClaw       ~/.openclaw/skills/panguard/SKILL.md

    All platforms configured.
    ```

    <Info>
      沒偵測到任何平台？沒關係，你還是可以直接用 CLI。跳到下一步就好。
    </Info>
  </Step>

  <Step title="掃描 AI Agent Skill">
    稽核你系統上所有已安裝的 MCP Skill 和 AI Agent 工具：

    ```bash
    panguard-guard scan
    ```

    掃描會用 61 條 ATR (Agent Threat Rules) 檢查所有已安裝的 Skill，涵蓋 prompt injection、tool poisoning、credential theft 等 9 大威脅類別：

    ```
    PANGUARD [#] AI  v1.3.0

    Scanning: claude-code, claude-desktop, cursor, openclaw, codex, workbuddy, nemoclaw, arkclaw

    Results:
      totalSkills: 23
      CLEAN:    18
      MEDIUM:    3
      HIGH:      1
      CRITICAL:  1

    highestRisk: HIGH
    ```

    加 `--json` 可輸出機器可讀格式，AI Agent 能直接解析。

    <Warning>
      **CRITICAL 和 HIGH 的發現需要你處理。** 逐一檢查每個發現，決定要保留、更新還是移除有問題的 Skill。Panguard 不會自動幫你刪東西。
    </Warning>
  </Step>

  <Step title="啟動即時防護">
    掃描是一次性檢查。要持續監控的話，啟動 Guard：

    ```bash
    panguard-guard start
    ```

    Guard 在背景執行，監控所有 AI Agent 的活動：

    ```
    PANGUARD [#] AI  v1.3.0

    Guard engine started.

    Status:     RUNNING
    Mode:       learning (Day 1/7)
    Rules:      61 ATR detection rules
    Dashboard:  http://localhost:3100
    ```

    <Info>
      **學習模式（第 1-7 天）：** Guard 觀察正常行為並建立基線。這段期間不會有誤報。第 7 天之後，異常偵測自動啟動。
    </Info>

    其他常用指令：

    ```bash
    panguard-guard status    # 檢查是否在執行
    panguard-guard stop      # 停止 daemon
    ```
  </Step>
</Steps>

## 兩個 CLI，不同用途

| 指令 | 功能 |
| --- | --- |
| `panguard scan` | 系統層級安全掃描（port、防火牆、服務） |
| `panguard-guard scan` | AI Agent Skill 掃描（MCP 工具、prompt injection、供應鏈） |
| `panguard setup` | 自動設定 AI 平台 |
| `panguard-guard start` | 啟動 AI Agent 即時監控 |

大多數情況你會先跑 `panguard-guard scan`（檢查你的 AI 工具），再跑 `panguard-guard start`（持續監控）。

## 集體防禦怎麼運作

跑 Panguard 的同時，你的機器就加入了集體防禦網路：

1. **Guard 在你的機器上偵測到威脅。**
2. **匿名 hash 分享到 Threat Cloud**（不含個資、不含原始碼）。
3. **3 個獨立掃描器確認**同樣的 pattern -- 自動產生新偵測規則。
4. **所有 Panguard 使用者在 1 小時內收到新規則。**

一台機器被攻擊，一小時後所有機器都免疫。
