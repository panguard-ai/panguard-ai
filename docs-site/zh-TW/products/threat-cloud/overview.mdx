Panguard Threat Cloud 是一個集體威脅情報平台。每個偵測到威脅的 Guard 代理都會將匿名化資料貢獻至雲端。每個代理都能從社群的集體偵測中受益。結果是：在一台機器上安裝 Panguard，就能獲得整個網路情報的防護。

## 運作方式

```
Agent A（偵測到威脅）──> 匿名化 ──> Threat Cloud ──> 所有 Agent 受益
Agent B（偵測到威脅）──> 匿名化 ──┘
Agent C（查詢）  ─────────────────────────┘
```

當 Guard 偵測到威脅時：
1. `ReportAgent` 匿名化威脅資料（IP /16 遮罩、無主機名稱、無檔案內容）
2. 匿名化的 IoC 提交到 Threat Cloud
3. Threat Cloud 驗證、關聯並儲存資料
4. 其他 Agent 在偵測管線中查詢 Threat Cloud，從集體情報中受益

## 核心功能

| 功能 | 說明 |
|------|------|
| **IoC 儲存** | IP 位址、網域、檔案雜湊、URL |
| **關聯引擎** | 跨來源關聯相關指標 |
| **攻擊活動追蹤** | 將相關 IoC 群組為攻擊活動 |
| **Feed 分發** | IP 和網域黑名單推送至 Agent |
| **目擊記錄** | 追蹤多少 Agent 看過每個指標 |
| **信譽引擎** | 基於目擊頻率和來源多樣性的信心評分 |
| **ATR 規則共享** | 社群貢獻的 AI Agent 威脅偵測規則 |
| **LLM 審查** | Claude Sonnet 4 自動審查提交的 ATR 提案 |
| **ATR 規則共享** | 社群貢獻的 AI Agent 偵測規則 |
| **稽核日誌** | 所有提交和查詢的完整稽核軌跡 |

## 快速開始

不需任何設定。Guard 啟動時自動連接公開 Threat Cloud：

```bash
panguard guard start --dashboard
```

Threat Cloud 同步每小時在背景自動進行。您的 Agent 貢獻匿名化威脅資料，並接收更新的規則、IoC feed 和黑名單。

<Info>
  **零設定。** 公開 Threat Cloud `tc.panguard.ai` 對所有 Panguard 使用者免費。標準 Agent 同步不需 API 金鑰。
</Info>

## 同步內容

每小時，Guard Agent 與 Threat Cloud 交換資料：

| 方向 | 資料 | 說明 |
|------|------|------|
| **上傳** | 匿名化威脅事件 | IP /16 遮罩、無主機名稱、無檔案內容 |
| **上傳** | ATR 提案 | 本地偵測到的新 AI Agent 威脅模式 |
| **上傳** | 技能審計結果 | 安全/不安全的技能指紋 |
| **下載** | ATR 規則 | 已確認的 AI Agent 威脅偵測規則 |
| **下載** | IP 黑名單 | 已知惡意 IP |
| **下載** | 網域黑名單 | 已知惡意網域 |

## IoC 類型

| 類型 | 範例 | 說明 |
|------|------|------|
| **IP 位址** | `203.0.113.50` | 已知惡意來源或 C2 伺服器 |
| **網域** | `evil.example.com` | 惡意網域或釣魚網站 |
| **檔案雜湊** | `sha256:abc123...` | 已知惡意軟體簽章 |
| **URL** | `http://evil.example.com/payload` | 惡意軟體散佈 URL |

## Feed 來源

Threat Cloud 聚合 11 個外部情報來源，包含：

| Feed | 資料類型 |
|------|----------|
| **ThreatFox** (abuse.ch) | 惡意軟體、C2、殭屍網路 IoC |
| **URLhaus** (abuse.ch) | 惡意軟體散佈 URL |
| **Feodo Tracker** (abuse.ch) | 殭屍網路 C2 伺服器 IP |
| **GreyNoise** | 大規模掃描與網路噪音 IP |
| **AbuseIPDB** | 社群回報的惡意 IP |

另有 6 個專門針對 AI Agent 威脅、供應鏈攻擊和區域威脅情報的 feed。

## ATR 社群管線

當 Guard 偵測到新的 AI Agent 威脅模式：

1. Guard 在本地起草 ATR 規則
2. 規則提案提交到 Threat Cloud（匿名化）
3. **Claude Sonnet 4** 審查提案的品質和準確性
4. 社群目擊確認或拒絕該模式
5. 確認的規則被推廣並分發給所有 Agent
