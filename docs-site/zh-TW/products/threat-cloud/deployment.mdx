每個 Panguard agent 自動連上公開 Threat Cloud。不用設定、不用 API key、不用帳號。

## 預設連線

啟動 Guard 就自動連 `tc.panguard.ai`：

```bash
panguard guard start --dashboard
```

就這樣。你的 agent 已經加入集體情報網路了。

## 自動做的事

| 間隔 | 動作 |
|------|------|
| **啟動時** | 從本機安裝載入內建 ATR 規則（61 條） |
| **每 1 小時** | 跟 Threat Cloud sync：拉新規則、傳匿名化威脅 |
| **偵測到威脅時** | 提交匿名化事件資料到 Threat Cloud |
| **skill 審計時** | 回報安全/不安全的 skill 指紋 |

## 自訂 endpoint

把 agent 指到別台 Threat Cloud：

```bash
panguard guard start --threat-cloud https://your-server:8080
```

或設環境變數：

```bash
export TC_ENDPOINT=https://your-server:8080
panguard guard start
```

或寫在 `~/.panguard/config.json`：

```json
{
  "threatCloudEndpoint": "https://your-server:8080",
  "threatCloudApiKey": "your-api-key"
}
```

## 離線模式

Guard 離線也能正常運作。如果 Threat Cloud 連不上：

- 內建的 61 條 ATR 規則照常偵測
- 本機 AI（Ollama）繼續做分析
- 事件在本機排隊，連線恢復後自動 sync
- 防護不會中斷

```bash
# 明確關掉 Threat Cloud sync
panguard guard start --no-cloud
```

## 驗證連線

看 agent 的 Threat Cloud 連線狀態：

```bash
panguard status --json
```

```json
{
  "threatCloud": {
    "connected": true,
    "endpoint": "https://tc.panguard.ai/api",
    "lastSync": "2026-03-13T08:00:00Z",
    "rulesReceived": 247,
    "threatsSubmitted": 12
  }
}
```

也可以在儀表板 `http://127.0.0.1:9100`（Threat Cloud 頁面）看。
