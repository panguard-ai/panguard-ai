每個 Panguard Agent 自動連接公開 Threat Cloud。不需設定、不需 API 金鑰、不需帳號。

## 預設連線

啟動 Guard 時，自動連接 `tc.panguard.ai`：

```bash
panguard guard start --dashboard
```

就這樣。您的 Agent 已加入集體情報網路。

## 自動執行的動作

| 間隔 | 動作 |
|------|------|
| **啟動時** | 從本地安裝載入內建 ATR 規則 |
| **每 1 小時** | 與 Threat Cloud 同步：下載新規則、上傳匿名化威脅 |
| **偵測到威脅時** | 提交匿名化事件資料到 Threat Cloud |
| **技能審計時** | 回報安全/不安全的技能指紋 |

## 自訂端點

將 Agent 指向不同的 Threat Cloud 實例：

```bash
panguard guard start --threat-cloud https://your-server:8080
```

或設定環境變數：

```bash
export TC_ENDPOINT=https://your-server:8080
panguard guard start
```

或在 `~/.panguard/config.json` 中設定：

```json
{
  "threatCloudEndpoint": "https://your-server:8080",
  "threatCloudApiKey": "your-api-key"
}
```

## 離線模式

Guard 完全支援離線運作。如果 Threat Cloud 無法連線：

- 內建 ATR 規則（61 條）持續運作
- 本地 AI（Ollama）處理分析
- 事件在本地排隊，恢復連線後同步
- 防護永不中斷

```bash
# 明確停用 Threat Cloud 同步
panguard guard start --no-cloud
```

## 驗證連線

檢查 Agent 的 Threat Cloud 狀態：

```bash
panguard status --json
```

```json
{
  "threatCloud": {
    "connected": true,
    "endpoint": "https://tc.panguard.ai/api",
    "lastSync": "2026-03-13T08:00:00Z",
    "rulesReceived": 247,
    "threatsSubmitted": 12
  }
}
```

或在儀表板 `http://127.0.0.1:9100`（Threat Cloud 頁面）查看。
