Panguard AI 不是一個單一工具，而是一個由多款產品組成的整合平台，每款產品專注於安全生命週期的特定階段。它們共同形成一個閉環：掃描您的系統、即時監控、通知相關人員、產生合規報告，並在您的設備群組之間共享情報。

## 產品如何協同運作

這些產品被設計為在分層防禦策略中互相配合：

```
Scan ──> Guard ──> Threat Cloud
```

| 工作流程 | 涉及的產品 | 描述 |
|----------|-------------------|-------------|
| **評估與防護** | Scan + Guard | 執行一次性掃描建立基準線，然後啟用 Guard 進行持續監控。 |
| **偵測與通知** | Guard | Guard 即時偵測威脅，並將淺顯易懂的通知傳送至您偏好的管道（Telegram、Slack、Email、Webhook）。 |
| **合規與報告** | Scan | 掃描發現項目直接對應合規控制措施。透過 `panguard scan --output report.pdf` 按需產生稽核就緒的 PDF 報告。 |
| **擴展與集中化** | Guard + Threat Cloud | 在您的設備群組中部署 Guard 代理，並透過 Threat Cloud 在全域關聯威脅。 |
| **AI 助手** | MCP + Scan + Guard | 在 Claude 或 Cursor 中使用自然語言來執行掃描、檢查狀態並透過 MCP 伺服器封鎖 IP。 |

## 架構概覽

Panguard 建構為由 pnpm workspaces 管理的 **13 套件 TypeScript monorepo**。所有套件共享一個通用的 `@panguard-ai/core` 基礎，提供規則引擎、監控引擎、AI 供應商抽象層、國際化和結構化日誌記錄。

| 層級 | 組件 | 職責 |
|-------|-----------|----------------|
| **端點** | Guard, Scan | 本地偵測、掃描和通知 |
| **管理** | Guard | 端點防護與本地設備群組協調 |
| **情報** | Threat Cloud | 集體 IoC 共享 |
| **整合** | MCP | AI 助手整合 |

<Info>
  每個產品都可以獨立運行。您不需要部署完整套件。從 `panguard scan` 開始，隨著需求增長再添加產品。
</Info>
