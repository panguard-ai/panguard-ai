## 支援的作業系統

| 作業系統 | 最低版本 | 架構 | 備註 |
|----|----------------|--------------|-------|
| macOS | 12 (Monterey)+ | x64、ARM64 (Apple Silicon) | 完整支援，包括 Guard 和 Trap |
| Ubuntu | 20.04 LTS+ | x64、ARM64 | 建議的 Linux 發行版 |
| Debian | 11 (Bullseye)+ | x64、ARM64 | 完整支援 |
| CentOS / RHEL | 8+ | x64 | 相容 SELinux |
| Windows | 10 (build 1903)+ | x64 | Guard 需要系統管理員權限 |

## 系統需求

<AccordionGroup>
  <Accordion title="Node.js">
    需要 **Node.js >= 20**。Panguard 使用現代 JavaScript 功能（top-level await、native fetch），需要 Node.js 20 或更高版本。

    檢查您的版本：
    ```bash
    node --version
    ```

    透過 [nvm](https://github.com/nvm-sh/nvm) 安裝或更新：
    ```bash
    nvm install 20
    nvm use 20
    ```

    或透過套件管理器：
    ```bash
    # macOS
    brew install node@20

    # Ubuntu/Debian
    curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
    sudo apt-get install -y nodejs

    # Windows (winget)
    winget install OpenJS.NodeJS.LTS
    ```
  </Accordion>
  <Accordion title="Root / 系統管理員權限">
    不同的 Panguard 模組有不同的權限需求：

    | 模組 | 需要 Root/Admin | 原因 |
    |--------|-------------------|--------|
    | `panguard scan` | 否 | 以使用者權限讀取檔案 |
    | `panguard guard` | **是** | 監控系統日誌、管理防火牆規則 |
    | `panguard trap` | **是** | 綁定到特權埠（< 1024） |
    | `panguard chat` | 否 | 透過 HTTPS 發送通知 |
    | `panguard report` | 否 | 從現有資料產生報告 |
    | `panguard manager` | 否 | 作為一般 HTTP 伺服器運行 |

    使用 `sudo` 執行特權模組：
    ```bash
    sudo panguard guard start
    sudo panguard trap start
    ```
  </Accordion>
  <Accordion title="磁碟空間">
    | 元件 | 大約大小 |
    |-----------|-----------------|
    | Panguard CLI + 核心 | 約 50 MB |
    | Sigma 規則（3,000+） | 約 15 MB |
    | YARA 規則（900+） | 約 8 MB |
    | Threat Cloud 資料庫（自架） | 100 MB -- 10 GB（視資料量而定） |
    | Guard 日誌 + 資料 | 隨時間增長，建議至少 1 GB |
  </Accordion>
  <Accordion title="網路">
    Panguard 需要對外的 HTTPS 存取：

    | 目的地 | 用途 |
    |-------------|---------|
    | `localhost:3000` | 身份驗證（自建部署） |
    | `tc.panguard.ai` | Threat Cloud 情報摘要 |
    | `registry.npmjs.org` | 套件安裝和更新 |
    | `api.anthropic.com` | AI 分析（如果使用 Anthropic） |
    | `api.openai.com` | AI 分析（如果使用 OpenAI） |
    | `localhost:11434` | Ollama 本機 AI（如果已設定） |

    所有連線可選擇性地透過 `HTTPS_PROXY` 環境變數經由 HTTP 代理。
  </Accordion>
</AccordionGroup>

## 如何更新

將 Panguard 更新至最新版本：

<CodeGroup>
```bash npm（全域安裝）
npm update -g panguard
```

```bash npx（免安裝）
npx panguard@latest doctor
```

```bash 驗證版本
panguard --version
```
</CodeGroup>

更新後，重新啟動任何運行中的服務：

```bash
# 重新啟動 Guard
sudo panguard guard stop
sudo panguard guard start

# 重新啟動 Chat
panguard chat stop
panguard chat start
```

<Info>
Panguard 會自動檢查更新，並在有新版本時在 CLI 中顯示通知。偵測規則（Sigma + YARA）獨立於 CLI 工具進行更新。
</Info>

## 如何解除安裝

若要從您的系統完全移除 Panguard：

<Tabs>
  <Tab title="macOS / Linux">
    ```bash
    # 1. 停止所有運行中的服務
    sudo panguard guard stop
    panguard chat stop

    # 2. 移除系統服務（如果已安裝）
    sudo panguard guard uninstall-service

    # 3. 解除安裝 npm 套件
    npm uninstall -g panguard

    # 4. 移除資料目錄
    rm -rf ~/.panguard
    rm -rf ~/.panguard-guard
    rm -rf ~/.panguard-chat

    # 5. 移除憑證
    rm -f ~/.panguard/credentials.json

    # 6. 移除 Guard 新增的防火牆規則
    # macOS:
    sudo pfctl -F all
    # Linux:
    sudo iptables -F PANGUARD 2>/dev/null
    sudo iptables -X PANGUARD 2>/dev/null
    ```
  </Tab>
  <Tab title="Windows">
    ```powershell
    # 1. 停止所有運行中的服務
    panguard guard stop
    panguard chat stop

    # 2. 移除 Windows 服務（如果已安裝）
    panguard guard uninstall-service

    # 3. 解除安裝 npm 套件
    npm uninstall -g panguard

    # 4. 移除資料目錄
    Remove-Item -Recurse -Force "$HOME\.panguard"
    Remove-Item -Recurse -Force "$HOME\.panguard-guard"
    Remove-Item -Recurse -Force "$HOME\.panguard-chat"

    # 5. 移除防火牆規則
    Remove-NetFirewallRule -DisplayName "Panguard*"
    ```
  </Tab>
</Tabs>

<Warning>
解除安裝會移除所有本機資料，包括掃描歷史、威脅日誌和設定。如果您需要歷史記錄，請先使用 `panguard report generate` 匯出您的資料。
</Warning>

## 常見錯誤訊息

<AccordionGroup>
  <Accordion title="Error: Node.js version 20 or later is required">
    您的 Node.js 版本太舊。更新至 Node.js 20+：
    ```bash
    nvm install 20 && nvm use 20
    ```
  </Accordion>
  <Accordion title="Error: EACCES permission denied">
    您在沒有 root 存取權限的情況下執行特權模組：
    ```bash
    sudo panguard guard start
    ```
    對於非 root 設定，請參閱[系統服務指南](/guides/system-service)。
  </Accordion>
  <Accordion title="Error: Cannot find module 'panguard'">
    Panguard 未全域安裝，或您的 PATH 未包含 npm 全域 bin 目錄：
    ```bash
    npm install -g panguard
    # 或檢查您的 PATH：
    npm config get prefix
    ```
  </Accordion>
  <Accordion title="Error: ECONNREFUSED">
    目標服務未運行或無法連線。請檢查：
    - 服務是否已啟動？（`panguard status`）
    - 埠是否正確？（檢查環境變數）
    - 防火牆是否阻擋了連線？
  </Accordion>
</AccordionGroup>
