## 登入時瀏覽器未開啟

執行 `panguard login` 時，CLI 會嘗試開啟您的預設瀏覽器進行身份驗證。如果瀏覽器未開啟：

<AccordionGroup>
  <Accordion title="使用 --no-browser 旗標">
    跳過自動瀏覽器啟動並手動登入：

    ```bash
    panguard login --no-browser
    ```

    CLI 會列印一個 URL。將其複製並貼上到您的瀏覽器中，然後在 CLI 提示中輸入回傳的 token。
  </Accordion>
  <Accordion title="SSH / 無介面環境">
    在沒有桌面環境的伺服器上，請始終使用 `--no-browser`：

    ```bash
    panguard login --no-browser
    ```

    或者，在有瀏覽器的機器上登入並複製憑證檔案：

    ```bash
    # 在有瀏覽器的機器上：
    panguard login
    cat ~/.panguard/credentials.json

    # 在伺服器上：
    mkdir -p ~/.panguard
    # 將憑證 JSON 貼上到：
    nano ~/.panguard/credentials.json
    chmod 600 ~/.panguard/credentials.json
    ```
  </Accordion>
  <Accordion title="WSL（Windows Subsystem for Linux）">
    WSL 可能無法啟動 Windows 瀏覽器。設定瀏覽器路徑：

    ```bash
    export BROWSER="/mnt/c/Program Files/Google/Chrome/Application/chrome.exe"
    panguard login
    ```

    或如上所述使用 `--no-browser`。
  </Accordion>
</AccordionGroup>

## 登入逾時

如果在您完成身份驗證前登入流程逾時：

```
Error: Login timed out. Please try again.
```

**解決方案：**

1. 重試登入 -- 預設逾時為 120 秒：
   ```bash
   panguard login
   ```

2. 檢查您的網路連線 -- CLI 必須連線到 `localhost:3000`

3. 如果在企業代理後面，設定代理環境變數：
   ```bash
   export HTTPS_PROXY=http://proxy.example.com:8080
   panguard login
   ```

## Token 已過期

```
Error: Authentication token has expired. Please log in again.
```

JWT token 在 **24 小時**後到期。重新驗證：

```bash
panguard login
```

<Tip>
對於自動化腳本和 CI/CD 管線，請使用 API token 而非 JWT token。API token 在明確撤銷前不會過期。從您的帳號儀表板產生一個，或使用 `panguard token create`。
</Tip>

## 憑證檔案權限

`~/.panguard/credentials.json` 中的憑證檔案儲存您的身份驗證 token。它應該只有您的使用者可以讀取。

<Warning>
如果憑證檔案的權限過於寬鬆（其他使用者可讀取），Panguard 將拒絕使用它並顯示安全警告。
</Warning>

**修復權限：**

<CodeGroup>
```bash macOS / Linux
chmod 600 ~/.panguard/credentials.json
```

```powershell Windows
icacls "$HOME\.panguard\credentials.json" /inheritance:r /grant:r "$($env:USERNAME):F"
```
</CodeGroup>

驗證：
```bash
ls -la ~/.panguard/credentials.json
# 應顯示：-rw------- 1 youruser yourgroup ... credentials.json
```

## 無效或損壞的憑證

如果您看到 `Error: Invalid token format` 或 `Error: Failed to parse credentials` 等錯誤：

```bash
# 移除損壞的憑證檔案
rm ~/.panguard/credentials.json

# 重新登入
panguard login
```

## Google OAuth 問題

如果 Google 登入失敗：

<AccordionGroup>
  <Accordion title="彈出視窗被封鎖">
    在您的瀏覽器設定中允許來自 `localhost:3000` 的彈出視窗。
  </Accordion>
  <Accordion title="重新導向 URI 不匹配">
    這是伺服器端的設定問題。如果您在 `localhost:3000` 上遇到此問題，請聯繫支援。
  </Accordion>
  <Accordion title="自架 auth server">
    確保 `GOOGLE_CLIENT_ID` 和 `GOOGLE_CLIENT_SECRET` 已正確設定，且 Google Cloud Console 中的重新導向 URI 與您的 auth server URL 匹配。
  </Accordion>
</AccordionGroup>

## 雙因素驗證 (2FA) 問題

<AccordionGroup>
  <Accordion title="遺失驗證器 app">
    使用您的備用代碼之一登入：
    ```bash
    panguard login
    # 當提示輸入 TOTP 代碼時，改為輸入備用代碼
    ```
    每個備用代碼為一次性使用。登入後，停用 2FA 並重新啟用以取得新的備用代碼。
  </Accordion>
  <Accordion title="TOTP 代碼不被接受">
    - 確認您的裝置時鐘已同步（TOTP 是基於時間的）
    - 確保您使用的是正確帳號的代碼
    - 代碼每 30 秒輪換 -- 嘗試等待下一個代碼
    - 如果問題持續，請使用備用代碼
  </Accordion>
</AccordionGroup>
