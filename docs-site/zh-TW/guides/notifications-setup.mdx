---
title: "設定通知"
description: "設定警報頻道，讓 Panguard 透過 Telegram、Slack、Email、LINE 或 Webhook 通知您威脅。"
icon: "bell"
---

Panguard Chat 將安全警報傳送至您團隊已在使用的頻道。本指南涵蓋設定 5 個支援的通知頻道，以及配置使用者角色以控制通知對象與內容。

<Steps>
  <Step title="初始化 Chat 設定">
    執行互動式設定精靈：

    ```bash
    panguard chat setup
    ```

    精靈將引導您完成頻道選擇與憑證輸入。您也可以使用下方的旗標個別設定頻道。
  </Step>

  <Step title="設定通知頻道">
    設定一個或多個通知頻道：

    <Tabs>
      <Tab title="Telegram">
        1. 在 Telegram 上訊息 [@BotFather](https://t.me/BotFather) 並建立新機器人
        2. 複製機器人 Token
        3. 與您的機器人開始對話並傳送任何訊息
        4. 從 Telegram API 取得您的 chat ID，或使用 [@userinfobot](https://t.me/userinfobot)

        ```bash
        panguard chat setup --channel telegram \
          --telegram-token "123456:ABC-DEF..." \
          --telegram-chat-id "-1001234567890"
        ```

        <Tip>
          若要群組通知，請將機器人加入 Telegram 群組並使用群組 chat ID（以 `-100` 開頭）。
        </Tip>
      </Tab>

      <Tab title="Slack">
        1. 前往 [api.slack.com/apps](https://api.slack.com/apps) 並建立新應用程式
        2. 在 **Bot Token Scopes** 下新增 `chat:write` OAuth 範圍
        3. 將應用程式安裝至您的工作區
        4. 複製 Bot User OAuth Token

        ```bash
        panguard chat setup --channel slack \
          --slack-token "xoxb-..." \
          --slack-channel "#security-alerts"
        ```
      </Tab>

      <Tab title="Email">
        設定 SMTP 憑證以接收電子郵件通知：

        ```bash
        panguard chat setup --channel email \
          --smtp-host "smtp.gmail.com" \
          --smtp-port 587 \
          --smtp-user "alerts@yourcompany.com" \
          --smtp-pass "app-password" \
          --email-to "security-team@yourcompany.com"
        ```

        <Info>
          Gmail 使用者請使用[應用程式密碼](https://support.google.com/accounts/answer/185833)而非帳號密碼。請先啟用雙因素驗證。
        </Info>
      </Tab>

      <Tab title="LINE">
        1. 在 [notify-bot.line.me](https://notify-bot.line.me/) 建立 LINE Notify Token
        2. 選擇接收通知的群組或一對一對話

        ```bash
        panguard chat setup --channel line \
          --line-token "your-line-notify-token"
        ```
      </Tab>

      <Tab title="Webhook">
        傳送原始 JSON 酬載至任何 HTTP 端點：

        ```bash
        panguard chat setup --channel webhook \
          --webhook-url "https://your-server.com/api/panguard-alerts" \
          --webhook-secret "your-hmac-secret"
        ```

        Panguard 傳送 `POST` 請求，酬載結構如下：

        ```json
        {
          "event": "guard.alert",
          "severity": "critical",
          "title": "Reverse shell detected",
          "details": { ... },
          "timestamp": "2026-03-07T14:22:01Z",
          "signature": "sha256=..."
        }
        ```

        `signature` 標頭包含使用您的 webhook 密鑰對 body 進行 HMAC-SHA256 計算的結果，用於驗證。
      </Tab>
    </Tabs>
  </Step>

  <Step title="設定使用者角色">
    Panguard 支援 3 種使用者角色，控制通知的詳細程度：

    | 角色 | 接收內容 | 適合 |
    | --- | --- | --- |
    | `developer` | 所有嚴重性等級，附完整技術細節 | 工程師、DevOps |
    | `boss` | 僅 CRITICAL 與 HIGH，白話語言摘要 | 高層主管、管理階層 |
    | `it_admin` | 所有嚴重性等級，中等細節，行動項目 | IT 營運 |

    為每個頻道設定角色：

    ```bash
    panguard chat setup --channel telegram --role developer
    panguard chat setup --channel email --role boss
    panguard chat setup --channel slack --role it_admin
    ```

    <Tip>
      您可以用不同角色多次設定相同頻道類型。例如，將 developer 等級的警報傳送至 `#security-engineering`，boss 等級的摘要傳送至 `#security-executive`。
    </Tip>
  </Step>

  <Step title="測試設定">
    向所有已設定的頻道傳送測試通知：

    ```bash
    panguard chat test
    ```

    ```
      PANGUARD AI - Chat Test

      Sending test notification...

      Telegram   ... sent
      Slack      ... sent
      Email      ... sent

      All channels verified.
    ```

    測試特定頻道：

    ```bash
    panguard chat test --channel slack
    ```
  </Step>

  <Step title="確認已啟用的頻道">
    列出所有已設定的通知頻道及其狀態：

    ```bash
    panguard chat status
    ```

    ```
      PANGUARD AI - Chat Status

      -- Channels -------------------------------

      Telegram   Active   role: developer   last sent: 2m ago
      Slack      Active   role: it_admin    last sent: 15m ago
      Email      Active   role: boss        last sent: 1h ago

      3 channels configured, 3 active.
    ```
  </Step>
</Steps>
