Panguard Threat Cloud is a collective threat intelligence platform. Every Guard agent that detects a threat contributes anonymized data to the cloud. Every agent benefits from the community's collective detections. The result: install Panguard on one machine and get the protection of the entire network's intelligence.

## How It Works

```
Agent A (detects threat) ──> Anonymize ──> Threat Cloud ──> All Agents benefit
Agent B (detects threat) ──> Anonymize ──┘
Agent C (queries) ─────────────────────────┘
```

When a Guard agent detects a threat:
1. The `ReportAgent` anonymizes the threat data (IP /16 masking, no hostnames, no file contents)
2. Anonymized IoCs are submitted to Threat Cloud
3. Threat Cloud validates, correlates, and stores the data
4. Other agents query Threat Cloud during their detection pipeline and benefit from the collective intelligence

## Quick Start

No configuration needed. Guard automatically connects to the public Threat Cloud when started:

```bash
panguard guard start --dashboard
```

Threat Cloud sync happens automatically every hour in the background. Your agent contributes anonymized threat data and receives updated rules, IoC feeds, and blocklists.

<Info>
  **Zero configuration required.** The public Threat Cloud at `tc.panguard.ai` is free for all Panguard users. No API key needed for standard agent sync.
</Info>

## What Gets Synced

Every hour, your Guard agent exchanges data with Threat Cloud:

| Direction | Data | Description |
|-----------|------|-------------|
| **Upload** | Anonymized threat events | IP /16 masked, no hostnames, no file contents |
| **Upload** | ATR proposals | New AI agent threat patterns detected locally |
| **Upload** | Skill audit results | Safe/unsafe skill fingerprints |
| **Download** | ATR rules | Confirmed AI agent threat detection rules |
| **Download** | IP blocklist | Known malicious IPs |
| **Download** | Domain blocklist | Known malicious domains |

## Guard Integration

The Guard `DetectAgent` queries Threat Cloud during event processing:

1. Extract indicators from the security event (source IP, destination domain, file hash)
2. Query Threat Cloud for matches
3. If matched, the indicator's confidence score contributes to the `AnalyzeAgent`'s weighted verdict
4. Threat intel matches have a fixed confidence contribution of 85

## ATR Community Pipeline

When your Guard detects a new AI agent threat pattern:

1. Guard drafts an ATR rule locally
2. Rule proposal is submitted to Threat Cloud (anonymized)
3. **Claude Sonnet 4** reviews the proposal for quality and accuracy
4. Community sightings confirm or reject the pattern
5. Confirmed rules are promoted and distributed to all agents
